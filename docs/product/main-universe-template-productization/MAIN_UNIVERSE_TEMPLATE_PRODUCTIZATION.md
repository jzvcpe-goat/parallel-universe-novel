# 主宇宙模板产品化 v1

## P0 目标

把已锁定的六大冻结式主宇宙模板转成可发现、可阅读、可创作、可运营的产品入口，并让前端页面、后端接口、人工确认项和模板参数保持同一套合同。

## 页面入口锁定

### 首页 `/`

首页承担商业冷启动，不展示内部系统说明。

- 第一主 CTA：开始阅读《灯塔之外》。
- 第二入口：进入书城，看到六大主宇宙模板专区。
- 作者入口：写同题开场，默认带入 `beacon-beyond` 模板。
- 禁止出现：后端、PRD、接口、memo、蒸馏、质量刹车、参数冻结等内部词。

### 书城 `/library`

书城承担发现和筛选。

- 顶部标识：主宇宙模板。
- 列表必须展示六大模板。
- 每张卡片有两个动作：阅读、用这个模板创作。
- 分类必须覆盖：玄幻、都市、悬疑、仙侠、言情、历史、科幻、连载、短篇。

### 阅读页 `/story?world={world_id}`

阅读页承担第一条体验闭环。

- 左侧：世界封面、章节索引、分支地图。
- 中间：浅色正文阅读面板，支持滚动和翻页。
- 右侧：阅读保存、故事状态、角色记忆、剧情节奏、选择影响。
- 每个主宇宙至少有一个开场章节和两个选择。
- 读者做出选择后必须看到：选择已记录、分支状态、角色记忆变化、下一幕入口。

### 创作页 `/create?template={template_id}`

创作页承担模板转创作。

- 用户先选择主宇宙模板，再输入故事种子。
- 默认从 URL `template` 参数读取模板。
- 提交创作会话时必须把模板 ID、标题、题材、开场前提、主角缺口、第一选择点写入 request context。
- 读者端不展示 memo 或蒸馏；创作页可展示“平台预置”但不解释训练过程。

### Studio `/studio`

Studio 承担运营和交接。

- 展示主宇宙模板管理。
- 每个模板展示开场前提、主角缺口、第一选择点、人工确认项。
- 内部机制、接口状态、发布检查只允许在 Studio 或交接文档出现。
- 未接能力必须标记为待接入或二期，不得进入读者端。

## 人工填写 vs 模型预置边界

### 人工必须确认

- 作品名和商业定位。
- 目标读者与付费预期。
- 主角姓名、身份细节和不可替代的情感伤口。
- 关键人物是否可死亡、是否可背叛、关系禁区。
- 内容禁区、尺度边界和平台合规要求。
- 首发模板优先级和推荐位。

### memo 小模型/冻结模板提供

- 题材结构。
- 情绪比例。
- 人物功能位。
- 冲突机制。
- 章节节奏。
- 爽点、虐点、悬疑点密度。
- 分支结局形状。

### 后端生成或持久化

- `world_id`、`world_version_id`。
- `reader_session_id`。
- 世界线状态、选择记录、角色记忆。
- 待处理片段、发布检查、确认提交记录。
- 订阅、权益和阅读进度。

## P0 不做

- 不做完整模型训练闭环。
- 不做复杂支付体系。
- 不做全量运营后台。
- 不新增六大模板之外的新宇宙。
- 不用单页视觉优化替代产品结构。
