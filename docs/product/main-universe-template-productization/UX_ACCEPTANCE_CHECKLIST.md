# 主宇宙模板产品化 UX 验收清单

## P0 通过定义

P0 通过不等于视觉漂亮，而是这条链路成立：

首页发现主宇宙 -> 进入书城 -> 打开模板第一章 -> 做出第一个选择 -> 选择被记录 -> 可以继续阅读或用该模板创作 -> Studio 能看到模板和发布状态。

## 首页 `/`

- [ ] 第一屏不是阅读正文，也不是创作表单。
- [ ] 第一主按钮是开始阅读旗舰宇宙。
- [ ] 首页能看出这是一个“可选择故事书城”，不是后台控制台。
- [ ] 首页有进入书城和写同题开场的入口。
- [ ] 首页不出现内部词：后端、PRD、接口、demo、质量刹车、memo、蒸馏、参数冻结。
- [ ] 大屏 1440x900 下，主标题一行或自然断行，不挤压按钮。
- [ ] 移动端 390x844 下，无横向滚动。

## 书城 `/library`

- [ ] 顶部明确“主宇宙模板”或同等产品概念。
- [ ] 六大模板全部可见或可通过筛选找到。
- [ ] 每个模板有：标题、题材、卖点、开场章节、选择数量。
- [ ] 每个模板有两个动作：阅读、用这个模板创作。
- [ ] 历史和科幻模板能通过分类筛选出来。
- [ ] 不展示未接入的后台能力。

## 阅读页 `/story?world={id}`

- [ ] 六个模板都能进入阅读页，不出现未开放兜底。
- [ ] 每个模板至少有一章开场和两个选择。
- [ ] 正文在浅色阅读面板内，支持上下滚动和翻页。
- [ ] 阅读区与左右栏视觉对齐，不漂浮错位。
- [ ] 做出选择后，右侧反馈出现选择影响、角色记忆、剧情节奏和保存状态。
- [ ] 读者端文案只围绕故事，不出现工程术语。
- [ ] 移动端左右栏可折叠，正文优先。

## 创作页 `/create?template={id}`

- [ ] 进入创作页时能看到当前主宇宙模板。
- [ ] 用户可以切换六大模板。
- [ ] 用户输入故事种子后，创作会话 context 带上 `main_universe_template`。
- [ ] 创作页引导包含人物、场景、世界规则、冲突推进和章节钩子。
- [ ] 明确哪些是用户要确认的设定，哪些是平台预置。
- [ ] 不把 Kimi、Moonshot、memo、蒸馏作为用户可见卖点。

## Studio `/studio`

- [ ] Studio 明确是创作者/运营工作台。
- [ ] 可看到六大主宇宙模板。
- [ ] 每个模板展示：开场前提、主角缺口、第一选择点、人工确认项。
- [ ] 发布检查和确认发布只出现在 Studio。
- [ ] 未接能力标记为待接入或二期，不进入读者端。
- [ ] Studio 可出现接口和参数，但不得反向污染读者页。

## 前后端一致性

- [ ] `main-universe-templates.v1.json` 中的每个 `id` 都能映射到前端 `worldTemplates`。
- [ ] 前端可见模板必须在后端 `GET /v1/library/worlds` 中存在或明确走本地降级。
- [ ] 阅读选择必须可映射到 `POST /v1/reader/continue` 或 `POST /v1/scene/advance`。
- [ ] 创作模板选择必须写入 `POST /v1/creator/dialogue/sessions` context。
- [ ] Studio 发布检查必须映射到 `POST /v1/quality/evaluate`。
- [ ] Studio 确认发布必须映射到 `POST /v1/canon/commit`。

## 视觉和设计系统

- [ ] 页面使用现有 shadcn/ui 分层：`ui` primitives、design-system patterns、page templates。
- [ ] 不在页面里继续堆一次性 Tailwind 大块逻辑。
- [ ] 关键按钮、卡片、面板使用统一圆角、边框、字体和状态色。
- [ ] 概念图只作为验收板，不作为字段或接口来源。

## 移交条件

全部 P0 项通过后，交给后端团队的包应包含：

- `MAIN_UNIVERSE_TEMPLATE_PRODUCTIZATION.md`
- `main-universe-templates.v1.json`
- `UNIVERSE_TEMPLATE_SCHEMA.md`
- `FRONTEND_BACKEND_MAPPING.md`
- `UX_ACCEPTANCE_CHECKLIST.md`
- 按交互顺序排列的概念图/验收图
- 最新构建产物或前端预览链接
- 校验命令输出摘要
