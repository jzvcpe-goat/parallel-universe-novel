# Narrative OKF Knowledge Layer

This folder applies the OKF-style idea to the fiction platform: knowledge is
stored as Markdown with simple YAML frontmatter so humans, agents and release
gates can read the same contract.

The cards here are not a second runtime. They point back to the existing
authoritative sources:

- `docs/product/rules/genre-runtime-rules.v1.json`
- `docs/product/rules/GENRE_CONSTRAINT_RULES.md`
- `docs/product/rules/GENRE_KERNEL_RULES.md`
- `docs/baseline/NarrativeOS_Quantum_Engineering_Contract_v3_Onboarding.md`

## Boundary

- Cards are internal-agent-readable, not end-user product copy.
- Cards must not include representative work names or recoverable source
  mappings.
- Public output still comes from public projection DTOs and UI copy, not these
  internal cards.
